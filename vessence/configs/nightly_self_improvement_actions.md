# Nightly Self-Improvement Actions

Last verified: 2026-04-18

Source of truth:
- Orchestrator: `agent_skills/nightly_self_improve.py`
- Job registry: `JOBS` in `agent_skills/nightly_self_improve.py`
- Cron schedule: `0 1 * * * /home/chieh/google-adk-env/adk-venv/bin/python /home/chieh/ambient/vessence/agent_skills/nightly_self_improve.py`

The nightly self-improvement orchestrator runs once per night at 1:00 AM.
Jobs run sequentially with per-job time budgets. If one job fails or times out,
the orchestrator records the result and continues to the next job.

## Actions

| Order | Job | Timeout | Script | Description |
|---:|---|---:|---|---|
| 1 | Auto-Commit WIP pre | 2 min | `agent_skills/auto_commit_wip.py` | Commits current uncommitted work locally before auditors run, using an `auto-commit:` prefix. This gives later jobs a committed baseline. It does not force-push. |
| 2 | Dead Code Auditor | 15 min | `agent_skills/dead_code_auditor.py` | Scans for unreferenced files, dead functions, and duplicate function bodies. It only auto-deletes conservative cases: old, small, unreferenced files under `agent_skills/` or `test_code/`. Other findings are written to `configs/dead_code_report.md`. |
| 3 | Code Auditor | 30 min | `agent_skills/nightly_code_auditor.py` | Picks one whitelisted module from `configs/auditable_modules.md`, creates an `auto-audit/...` branch, asks Claude Opus to generate stress tests, attempts fixes, and commits only if tests pass. Failures are logged to `configs/audit_failures.md`. |
| 4 | Pipeline Audit, 30 prompts | 20 min | `agent_skills/pipeline_audit_100.py --n 30` | Pulls recent real prompts from `jane_prompt_dump.jsonl`, runs them through the live Stage 1/2/3 pipeline, judges stage behavior, auto-adds clear Stage 1 correction exemplars to Chroma, and logs Stage 2/3 issues to `configs/pipeline_audit_report.md`. |
| 5 | Doc Drift Auditor | 5 min | `agent_skills/doc_drift_auditor.py` | Compares docs and registries against actual cron, files, pipeline classes, class packs, and skill files. Safe fixes are committed with `auto-doc-sync:`. Ambiguous drift is written to `configs/doc_drift_report.md`. |
| 6 | Transcript Quality Review | 20 min | `agent_skills/transcript_quality_review.py` | Reads prompt dump, server pipeline logs, and Android diagnostics for the previous day. Codex identifies odd or wrong turns and traces causes from logs. Claude then validates findings against code, writes fixes and tests where valid, and appends to `configs/transcript_review_report.md`. |
| 7 | Memory Janitor | 30 min | `memory/v1/janitor_memory.py` | Waits for safe system load, backfills missed thematic archival, deduplicates cross-session themes, purges expired or old short-term memory, consolidates redundant long-term memories with Claude Opus, purges old logs, verifies code-related memories against the real codebase, and refreshes dynamic query markers. |
| 8 | Auto-Commit + Push post | 2 min | `agent_skills/auto_commit_wip.py --push` | Commits any reports, tests, or code fixes generated by the nightly jobs, then pushes to the remote. Push failure is logged but treated as non-fatal after local commit. |
| Final | Readable latest report | n/a | built into `agent_skills/nightly_self_improve.py` | Writes `configs/self_improvement_latest.md` with one readable section per stage: what it did, problems it found, improvements it made, and evidence files. Also archives a timestamped copy under `/home/chieh/ambient/vessence-data/reports/self_improvement/`. |

## Logs

- Orchestrator log: `/home/chieh/ambient/vessence-data/logs/self_improve_orchestrator.log`
- Per-job logs: `/home/chieh/ambient/vessence-data/logs/self_improve_<script_name>.log`
- Summary log: `configs/self_improve_log.md`
- Readable latest report: `configs/self_improvement_latest.md`
- Readable report archive: `/home/chieh/ambient/vessence-data/reports/self_improvement/`
- Vocal rollup: emitted via `agent_skills.self_improve_log`

## Notes

- A job marked `ok` may still have skipped substantive work. For example,
  `nightly_code_auditor.py` exits cleanly when the working tree is not clean.
- `transcript_quality_review.py` can produce code/test changes and a report even
  if it exits nonzero because the Claude validation/fix subprocess timed out.
- If the orchestrator log appears duplicated, check whether `nightly_self_improve.py`
  is both printing and appending to the same file while cron also redirects stdout
  to that file.
